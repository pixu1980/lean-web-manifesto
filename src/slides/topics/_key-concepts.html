Quindi cos'è il Lean Web?

È una proposta: un cambio di paradigma culturale nello sviluppo del web...

Definisci il concetto di Lean Web spiegando come sia nato dall'applicazione dei principi della Lean Manufacturing allo sviluppo software. Sottolinea che il Lean Web si concentra sull'eliminazione degli sprechi, sull'ottimizzazione delle risorse e sul miglioramento continuo delle performance. Fornisci un breve confronto con approcci tradizionali più complessi per evidenziare il valore dell'approccio lean.

...o forse giusto un po' di consigli in merito.

Princìpi del Lean Web

  • Riduzione del peso dei prodotti digitali
  • Ottimizzazione delle performance
  • Accessibilità e inclusività
  • Sostenibilità digitale
Spiega ciascuno dei concetti chiave del Lean Web con esempi pratici. Ad esempio, per la riduzione del peso delle pagine, menziona come eliminare librerie non necessarie possa migliorare i tempi di caricamento.

Il Costo di JavaScript

JavaScript è la parte più "costosa" dello stack tecnologico in termini di performance.

"Byte-for-byte, JavaScript is still the most expensive resource we send to devices, because it can delay interactivity in large ways."
Addy Osmani
(post del 2018, update talk del 2023)
Mostra come l'uso eccessivo di JavaScript possa rallentare le applicazioni, specialmente sui dispositivi mobili. Cita Addy Osmani per evidenziare l'impatto delle risorse JavaScript sulle performance. Fornisci dati o benchmark che dimostrano come la riduzione del JavaScript possa migliorare i tempi di risposta e l'interattività delle applicazioni.

Il "mito" del Virtual DOM

Il Virtual DOM sicuramente rappresenta un'utile strategia per ottimizzare gli aggiornamenti dell'interfaccia in applicazioni complesse

Spiega quando il Virtual DOM è davvero utile e quando può essere evitato. Confronta le performance di applicazioni che utilizzano il Virtual DOM con quelle che usano metodi di rendering più semplici. Sottolinea che, in molti casi, la complessità introdotta dal Virtual DOM non è giustificata dai benefici.

In molti casi questo approccio introduce un overhead non necessario in contesti dove la manipolazione diretta del DOM è già sufficientemente performante

Come sempre, la risposta è Dipende!

Se gli aggiornamenti sono minimi, se il sistema è configurato per un rendering diretto efficiente, affidarsi al Virtual DOM potrebbe risultare più pesante che vantaggioso.

Kyle Shevlin - Speaker & Mentor @ egghead.io
